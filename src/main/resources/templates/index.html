» Set app environment

The welcome headline can be customized with an environment variable. Use cf set-env GREETING ' ... ' to replace the Hello, world! in the headline with a customized message.

To apply the new app environment, run , wait until the staging process finishes and refresh this page.

» Monitor your app

Use to show some information like memory and compute load of your app. You can use the "Do something" button below to generate some load.

Use to tail the application logs or to view recent logs of your app. Again, you can use the "Do something" button to generate logs.

Use to ssh into your container. (Maybe try to find the java process, kill it and see what happens.)

Use to list events for your application (e.g. starts, restarts & crashes).

This app also provides a http endpoint that can be used by the runtime to monitor the app health status. To enable the health check, edit the manifest.yml, uncomment the lines with health-check-type: http and health-check-http-endpoint: /hc, save the file and run cf push again. After the app is successfully restarted, refresh this page.

Congratulations, your app is now monitored with a http health check. Try use the "Set unhealthy" button and use , and to trace what's happening.

» Scale your app

Maybe you noticed that your app became unavailable for a few seconds when it was killed or became unhealthy. To mitigate that problem - or if there is just too much traffic on your app for a single instance to handle it - you have to scale your app horizontally. Use to start another instance of your app.

After scaling your app horizontally, refresh this page every few seconds and see if something changes to the right. Again, run , and to trace what's happening. Again, use the "Do something" button below, to simulate some load.

In case your application requires more memory to run properly, you can also scale vertically. You could use the CLI again to scale give your application additional memory. However, we now want to use the manifest.yml file to set both memory and the number od instances so that will not be overridden by another cf push. To scale up, edit the manifest.yml, set instances: 2 and memory: 2048M, save the file and run cf push again. After the app is successfully restarted, refresh this page.

» Integrate services

In the beginning we changed the greeting message with an environment variable. Unfortunately, we need to restart every time the environment variables change. As this app cannot use any persistent file storage, we would like to integrate a data service.

Use cf marketplace to see which services are available in your environment. To demonstrate integration of services, we want to use Redis to store the greeting message.

Use cf create-service my-service p-redis shared-vm to create a service instance. Use to use the created service instance in you app. Use to restart your app and refresh this page to see what happens.

» Update your app

Unfortunately, there might be a bug in your app. You can check that by hitting the "Do something" button below.

To resolve the bug, open the Java class com.github.tscn.cf.Controller and fix the method complete. If you're confident that you have found and fixed the bug, rebuild the app with gradlew build.

To facilitate a zero-downtime deployment, we do not push the existing app. Instead, we push another app and do a blue-green-deployment by mapping and un-mapping routes.

Use to rename the existing app. Now you can push your new version with cf push --no-route. By using --no-route we can disable automatic route mapping for the app.
